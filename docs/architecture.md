# アーキテクチャ

## 目的

このリポジトリは、日常的に使うローカル開発環境を再現可能な形で管理するための dotfiles 集です。単に設定ファイルを置くだけではなく、セットアップ、CLI の公開、タスク実行、AI ツール設定まで含めて一貫した構成で管理します。

## 大きな責務分割

- `README.md`
  - 人向けの入口
- `AGENTS.md`
  - エージェント向けの入口
- `docs/`
  - リポジトリ全体の設計判断の正本
- 各サブシステム直下の policy
  - ローカルな詳細規約の正本

## 主要ディレクトリの役割

- `nvim/`
  - Neovim の設定とプラグイン定義
- `terminal/`, `ghostty/`, `wezterm/`, `zellij/`, `starship/`
  - 端末・シェル・プロンプト関連の設定
- `vscode/`
  - VS Code の設定
- `scripts/local/`
  - セットアップやリポジトリ内で使うローカルスクリプト
- `scripts/global/`
  - `~/.local/bin` と `mise` task に公開する単独実行コマンドの正本
- `scripts/utils/`
  - shell 用の共通補助スクリプト
- `homebrew/`
  - Homebrew パッケージ定義
- `bun/`
  - Bun グローバルパッケージ定義
- `ai/`
  - AI ツール設定と独自 skill
  - skill の応答言語は原則として依頼文の言語に合わせる
  - ただしコミットメッセージのように成果物自体の言語要件がある場合はその要件を優先する

## 設計原則

### 1. 正本を一箇所に寄せる

- セットアップや日常運用の正本は `README.md`
- リポジトリ全体の設計原則の正本は `docs/`
- サブシステム固有ポリシーの正本は実装の近く
- `AGENTS.md` は正本のコピーではなく参照先の案内に徹する

### 2. 実装と運用の境界を明確にする

- 単独実行コマンドは `scripts/global/` に置く
- リポジトリのセットアップ処理は `scripts/local/` に置く
- 共通処理は `scripts/utils/` に置く
- `mise` は task の入口と一覧のために使う

### 3. 局所ルールは局所に置く

- Neovim のキーバインドや表示方針は `nvim/lua/policies/` に置く
- リポジトリ全体の docs には要約とリンクだけを置く
- 新しい関数には、役割がひと目で分かる短い英語の doc comment を付ける

### 4. 機械可読な運用を優先する

- 反復的なセットアップは task 化する
- global command は公開経路をそろえる
- formatter と pre-commit を使って揺れを抑える

## ドキュメントの階層

1. `README.md`
2. `AGENTS.md`
3. `docs/*.md`
4. `*/policies/*.md`

上に行くほど入口に近く、下に行くほど詳細になります。
